Setup Guide

Complete all steps in order. Each step builds on the last — do not skip ahead.

Step 1 — Update & install system packages

bash
sudo apt update && sudo apt upgrade -y
sudo apt install -y tesseract-ocr libcap-dev i2c-tools
✓
Install should finish without any ERROR lines before continuing.

Step 2 — Clone repo & set up virtual environment

bash
git clone https://github.com/s4r131/AeroClean.git
cd AeroClean
python3 -m venv aeroclean_env
source aeroclean_env/bin/activate
pip install -r requirements.txt
⚠️
Reactivate with source aeroclean_env/bin/activate in every new terminal. Your prompt should show (aeroclean_env) before running anything.

Step 3 — Verify USB camera

The OV2311 is UVC-compliant — no config.txt changes or reboot needed. Run this before plugging in the camera, then again after — the new entry that appears is your device:

bash
ls /dev/video*   # run before plugging in — note existing entries
# plug in camera
ls /dev/video*   # new entry = your camera (e.g. /dev/video0 → camera_device: 0)
✓
Note the device number — it goes in config.json → camera_device (_s1). Default is 0. Only change it if other USB video devices are already connected.

Step 4 — Enable UART0 (ArduPilot FC)

Add to /boot/firmware/config.txt, save, then reboot:

/boot/firmware/config.txt
enable_uart=1

After reboot, run pinctrl -p and confirm pins 8 and 10 changed from none to TXD0 / RXD0. Then list devices:

bash
ls -l /dev/ttyAMA*
# Note the FC path → goes in config.json: mission.mavlink_uart

Step 5 — Enable range sensor interface

Add a free overlay UART to config.txt, save, then reboot:

/boot/firmware/config.txt
dtoverlay=uartX   # e.g. dtoverlay=uart4

After reboot, verify the new device and confirm the sensor is transmitting:

bash
ls -l /dev/ttyAMA*
cat /dev/ttyAMAX   # garbled binary = sensor is live. Ctrl+C to stop

Note the path → goes in config.json → range_sensor.uart.

Add to config.txt, save, then reboot:

/boot/firmware/config.txt
dtparam=i2c_arm=on

After reboot, verify the sensor is visible on the I2C bus:

bash
sudo i2cdetect -y 1
# Expected: 29 appears at address 0x29

Step 6 — Dialout group

bash
sudo usermod -aG dialout $USER
# Log out and back in, then verify:
groups   # expected output includes: dialout

On Pi OS Trixie, /dev/ttyAMA* devices are owned root:root by default — add a udev rule to permanently fix the group:

bash
echo 'KERNEL=="ttyAMA[0-9]*", GROUP="dialout", MODE="0660"' | sudo tee /etc/udev/rules.d/99-ttyama.rules
sudo udevadm control --reload-rules
sudo udevadm trigger
ls -la /dev/ttyAMA*   # expected: group column shows dialout

Step 7 — Configure config.json

Enter the device paths noted in steps 4–5. The mission will not start until all null values are filled in.

config.json
"mavlink_uart": "/dev/ttyAMA0",
"range_sensor": { "uart": "/dev/ttyAMAx" }
💡
pinctrl -p shows live GPIO state — pins change from no … = none to an alternate function after enabling interfaces. Run pinout to map GPIO numbers to physical pin positions on the header.
🛑
Disable serial login shell — do this last. Run all hardware tests on the Checklist screen first. The Final step card there has the commands. Do not disable the console before testing — you need it to diagnose wiring problems.

Model 1 — OCR Pipeline

Finds the word "dirty" written in marker on the board surface using Tesseract OCR with a preprocessing chain.

Preprocessing stages

1
Greyscale
Strips colour — OCR only needs luminance
2
CLAHE
Contrast Limited Adaptive Histogram Equalisation — lifts faint marker strokes that the board's reflective surface washes out
3
Adaptive threshold
Converts to black-and-white using local pixel neighbourhoods — handles shadows and hotspots a global threshold misses
4
Morphological close
Joins broken letter strokes so Tesseract sees whole characters instead of fragments

Tesseract config

python
# Neural-network engine, uniform text block
config = '--oem 3 --psm 6'
data = pytesseract.image_to_data(frame, config=config)

Result is searched for dirty (case-insensitive regex). Matches get red bounding boxes; clean frames show a green CLEAN banner.

Performance settings

Throttle interval
ocr_interval_s: 2.5 — Tesseract runs at most every 2.5 s. Last result redrawn between runs. Significantly reduces CPU load.
Downscale before OCR
ocr_scale: 0.5 — frame resized to 960×540 before preprocessing. Tesseract reads reliably at half resolution with ~4× less pixel data.
Board ROI crop
board_roi: [x, y, w, h] — optional crop to board area only. Limits Tesseract to the region of interest, cutting processing time further.
💡
Both throttle and downscale are active by default. Adjust in config.json if detection is too slow or misses text.

Run OCR mode

bash
python main.py --model ocr
# Test on a saved image (no Pi camera needed)
python main.py --model ocr --source board.jpg --once

Tips for a good dataset

💡

Vary lighting

Overhead fluorescent, natural window light, lamp at an angle — the Pi will encounter all of these.

🖊️

Vary board coverage

Lightly smudged, heavily marked, ghost marks, partially erased — capture the full spectrum.

📐

Vary angle slightly

Straight-on plus small tilts (±15°). Match the mounting height you'll use in production.

📏

Match production distance

Capture from the same distance the drone will be during SCAN so the model learns the right scale.

Label & Train

Step 1 — Label in Label Studio

1.Install and launch: pip install label-studio && label-studio start
2.Create project → Object Detection with Bounding Boxes
3.Add labels clean_board then dirty_board — order matters (IDs 0, 1)
4.Import images/raw/ — draw one tight bounding box per image
5.Export → YOLO format → extract to images/ and labels/
🎯
One box per image. Tight around the board surface. Pick a rule for ambiguous boards (e.g. "any residue = dirty") and apply it consistently.

Labeling rules

✓ Do
  • One bounding box per image — tightly around the entire board surface
  • Draw to board edges, not around individual marks
  • Decide a rule for ambiguous boards and apply it every time
  • 100–200 images minimum per class
✗ Don't
  • Draw multiple boxes per image
  • Draw a box around individual marks on the board
  • Leave board edges cut off outside the box
  • Mix labeling rules mid-dataset

Class decision guide

Board conditionLabel as
Fully erased, no visible residueclean_board
Ghost marks (faded but visible)dirty_board
Partial erasure — some marks remaindirty_board
Heavy writing / smudgingdirty_board
Smudge from eraser only — no inkclean_board
ℹ️
Class order matters — add clean_board first (ID 0), then dirty_board (ID 1). Swapping them changes class IDs in every label file and breaks training.

Required split

StateWhat happens
IDLEArms the FC and initiates takeoff to takeoff_altitude_m
SCANSlow yaw spin at scan_yaw_rate_dps; YOLO runs every frame looking for dirty_board
APPROACHPhase 1 — Align: holds position, corrects lateral/vertical until board is within align_threshold_px of frame centre. Phase 2 — Approach: drives forward at vx = kp_forward × (dist − stop_dist); stops when range sensor reads ≤ approach_stop_dist_m
CLEANHolds position; activates pump for pump_duration_s seconds, then actuates the wiper arm sweep
RETURNSwitches ArduPilot to RTL mode; waits for landing
DONE / ABORTEDTerminal states — all subsystems shut down cleanly
